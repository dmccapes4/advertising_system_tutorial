Module 1 — Indentation, scope, unreachable code

return exits the function. If every branch returns, lines below are dead code — even when they look like the happy path.

Bad (teaching example — do not copy)

unreachable return 
def process_refund(order_id: str) -> dict:

    customer = get_customer(order_id)

    if customer["balance"] < 0:

        return {"status": "denied"}

    else:

        return {"status": "error", "message": "account on hold"}

    return {"status": "approved"}  # never runs

Both if and else return → bottom line is a fossil.

Fixed — guard clauses + fall-through happy path

reachable 
def process_refund(order_id: str) -> dict:

    customer = get_customer(order_id)

    if customer["balance"] < 0:

        return {"status": "denied"}

    if customer["on_hold"]:

        return {"status": "error", "message": "account on hold"}

    return {"status": "approved"}  # runs when neither guard fired

Good — variables, elif warning, fall-through approve

reachable 
def process_refund(order_id: str) -> dict:
    customer = get_customer_by_order_id(order_id)
    order = get_order(order_id)
    balance = customer["balance"]
    order_price = order["total"]
    remaining = balance - order_price
    notice = ""

    if balance < order_price:
        return {"status": "denied", "shortfall": order_price - balance}
    elif remaining < 50:
        notice = "Low balance after refund — flag for review."
    else:
        log("this is an unnecessary condition")  # else optional — see md

    return {"status": "approved", "notice": notice}

elif sets notice but does not return — execution falls through to one shared approve.
