Module 2 — JSON first

Learn one thing: JSON. SQL is older and more important — and more tedious. JSON is easy: { = object, "key": value inside.

Book object

{
  "title": "Breakthrough Copy",
  "topics": ["copywriting", "hooks"]
}

Tool use — nested object

{
  "type": "tool_use",
  "name": "advertising_knowledge",
  "input": { "query": "fair use in headlines" }
}

Session log — agentic systems

No session_log → runaway loop. An unbounded while tool_use eats RAM until the OS kills the process. session_log.json / .jsonl = audit trail + debug + don't repeat the same mistake. TartaurusLoop writes one every run.

Log = story of the run · Checklist = resume · Result = deliverable
