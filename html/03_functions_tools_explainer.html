Module 3 — Tools: import a function vs run a file

Path A — import

from advertising_knowledge import advertising_knowledge

result = advertising_knowledge("fair use")

Agent / MCP calls the def. __main__ block does not run.

Path B — run file

python advertising_knowledge.py "fair use"

if __name__ == "__main__":
    ...

CLI / operator. __name__ == "__main__" → script entry.

Same .py file, two doors. MCP tools = Path A. Terminal smoke test = Path B.

MCP — expose the function to Claude Desktop / Cursor

@mcp.tool()
def advertising_knowledge(query: str) -> dict:
    return _advertising_knowledge(query)

User asks → who runs what?

1. Host sends user message + tool list to Claude
2. Claude returns tool_use (request — no Python yet)
3. Host calls MCP tools/call → your server.py runs the function
4. Host sends tool_result back to Claude
5. Claude end_turn → user sees the answer
